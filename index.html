Git and GitHub

Git

Git is a distributed version control system used primarily for managing and tracking changes in source code during software development. It allows multiple developers to collaborate on a project simultaneously and keeps track of all changes made to the codebase over tiem.

Here's a breakdown of how Git works and how to use it:

  1. Repository: A Git repository is essentially a folder that contains all the files and folders for your project, along with the version history of those files.
  2. Commits: A commit is a snapshot of the changes you've made to your files at a particular point in time. Each commit has a unique identifier and a commit message describing the changes made.
  3. Branches: Branches in Git allow you to work od different features or fixes in isolation from each other. YOu can create a new branch from an existing one, make changes, and then merge those changes back into the main branch.
  4. Merge: Merging is the process of combining the changes from one branch into another. This is typically done when you've finished working on a feature branch and want to incorporate thos changes into the main branch.
  5. Pull request: In Git-based collaboration workflows like GitHub, a pull request is a way to propose changes to a repository. It allows other developers to review your code, provide feedback, and ultimately merge your changes into the main branch.

To commit your changes to your GitHub repository from your local machine, you'll typically follow these steps using Git commands:

  1. Add files to staging area: First, you need to add the files you want to commit to the staging area. You can do this using the git add command followed by the file names or a dot (.) to add all files in the current directory. 
          
        git add <file1> <file2> ...
          # or
          git add .
  2. Commit changes: Once the files are staged, you can commit them to your local repository along with a commit message using the 'git commit' command: 
        
      git commit -m 'Your commit message here'
    Replace 'Your commit message here' with a brief, decsriptive message summarizing the changes you made in this commit.
  3. Push changes to GitHub: Finally, you'll push your commited changes from your local repository to your GitHub repository using the 'git push' command. 
        
      git push origin main

This sequence adds all changes in your current directory to the staging area, commits those changes with a commit message, and then pushes those changes to the 'main' branch of your GitHub repository. Make sure to replace 'main' with the actual branch name if you're working on a different branch.
